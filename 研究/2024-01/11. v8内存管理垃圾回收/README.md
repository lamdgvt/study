## V8 引擎
```
js引擎 v8引擎
V8 采用的是 解释和编译两种方式的混合使用 JIT技术
```

#### V8内存
```
V8 申请内存, 常驻内存集合
内存分为 堆和栈

栈一般存放 基本类型和引用类型的指针 (不需要垃圾回收) (会溢出)
堆一般用于 执行上下文(AO)时函数内的对象 (复合类型) (不会溢出)

当每个上下文(AO)执行期间 (地狱回调时), 
会产生局部上下文(AO), 也会进行堆栈的存放
当执行完当前的上下文(AO), 会自动进行栈释放. (闭包除外)(垃圾回收一般指的是堆  堆回收)
当局部上下文(AO) 执行到一定的程度时, 可能会有栈溢出 (破栈) 的情况 (地狱回调太多层, 递归死循环)
```

#### 垃圾回收
```
1. 不再使用的变量
2. 释放其占用的内存
3. 固定时间周期性运行

标记清除:
每次进入一个函数时 
会标记 "进入环境",
执行完一个函数时,
会标记 "离开环境".
当每次执行到 "离开环境" 标记时, 会清除内部的变量. 除了全局变量和闭包AO的变量.

引用计数:
引用次数如果为0, 就将被回收. 
引用不为 0 就不回收.
赋值 null 为 "解除引用"
```
