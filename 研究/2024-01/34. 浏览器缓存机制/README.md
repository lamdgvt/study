### 浏览器缓存机制

```
浏览器缓存机制 一般分为两种:
强缓存 和 协商缓存  主要目的是减少带宽压力,减少重复请求静态资源.

强缓存判断:
    1.  if  (不请求、network from cache 假请求标识)
            判断 cache-control: max-age: 毫秒数  本地缓存
        else 
            cache-control: no-cache (不缓存策略) 进行协商缓存判断  
        {
            
            协商缓存:  (发送真请求)
            if 服务端判断资源标识 是否一致
                一致, 将返回 304, 不返回资源
            else
                不一致 返回 200, 获取新资源
        }
```
